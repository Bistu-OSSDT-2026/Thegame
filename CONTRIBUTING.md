## 开发流程

我们采用 **分支开发 + Pull Request** 的工作流：

```
main 分支（主线，受保护）
  ↑
  ├── feature/snake-move     ← 开发蛇移动
  ├── feature/snake-ui       ← 开发界面
  ├── feature/ci-setup       ← 配置 CI
  └── ...
```

### 步骤

1. **领取任务**：从 Issue 列表中认领自己对应的任务
2. **创建分支**：从 `main` 分支拉取最新代码，创建自己的功能分支
3. **开发 & 提交**：在分支上完成开发，提交代码
4. **推送 & 创建 PR**：推送到远程仓库，创建 Pull Request
5. **等待审核**：綦竞尧下载代码测试，审核
6. **修改 & 合并**：根据审核意见修改，通过的合并到 `main`

---


> ⚠️ **禁止直接提交到 `main` 分支！**

## 提交信息规范

每次 `git commit` 请写明做了什么，格式如下：

```
[类型] 简要描述

示例：
[feat] 实现蛇的移动与身体增长逻辑
[fix] 修复食物刷新到蛇身上的问题
[docs] 更新 README 安装说明
[style] 统一代码缩进格式
[ci] 配置 GitHub Actions 自动检测
```

类型说明：

| 类型 | 含义 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档修改 |
| `style` | 代码格式调整 |
| `refactor` | 代码重构 |
| `test` | 测试相关 |
| `ci` | CI / 自动化配置 |

## Pull Request 规范

### PR 标题格式

```
[类型] 功能名称
```

示例：`[feat] 实现键盘控制蛇转向`

### PR 描述模板

请在 PR 中填写以下内容：

```markdown
## 📝 改动说明
简要描述做了什么改动

## 🔗 关联 Issue
Closes #3

## ✅ 自测结果
- [ ] 代码能正常运行
- [ ] 功能符合预期
- [ ] 没有引入新的报错

```

### PR 审核流程

1.綦竞尧审核通过
2. 所有 CI 检查通过（无红色 ❌）
3. 由綦竞尧最终合并到 `main`

## 代码风格

- 使用 **2 空格缩进**
- 变量名使用 **小驼峰命名法**（`snakeBody`、`foodPosition`）
- 常量使用 **大写下划线**（`GAME_SPEED`、`CANVAS_SIZE`）
- 函数名使用 **小驼峰命名法**（`moveSnake()`、`checkCollision()`）
- 每行代码不超过 **80 个字符**
- 关键逻辑必须写 **注释**

### 示例

```javascript
// 蛇的当前移动方向
let direction = 'right';

// 检查蛇头是否撞到墙壁
function checkWallCollision(head) {
  return head.x < 0 || head.x >= CANVAS_WIDTH ||
         head.y < 0 || head.y >= CANVAS_HEIGHT;
}
```

## Issue 使用规范

### 报告 Bug

发现 Bug 时，请到 GitHub Issues 页面点击 **New Issue**，选择 Bug 模板，填写：

- **Bug 描述**：发生了什么
- **复现步骤**：怎么触发这个 Bug
- **预期行为**：应该是什么样的
- **实际行为**：实际发生了什么
- **截图**：如果能截图请附上

### 请求新功能

有想加的功能？同样创建一个 Issue，描述：

- **功能描述**：想实现什么
- **使用场景**：为什么需要这个功能
- **实现思路**（可选）：你有什么想法

## 测试要求

### 开发者自测

提交 PR 之前，请至少完成以下自测：

- [ ] 在浏览器中打开 `index.html`，游戏能正常启动
- [ ] 自己负责的功能运行符合预期
- [ ] 没有浏览器控制台报错（按 F12 查看 Console）
- [ ] 不影响其他已有功能

### 测试员验证

綦竞尧在审核 PR 时，需要额外验证：

- [ ] 下载 PR 分支代码到本地运行
- [ ] 按 Issue 中的验收标准逐项测试
- [ ] 检查是否有性能问题（卡顿、内存泄漏）
- [ ] 在不同浏览器中测试（Chrome / Firefox / Edge）
